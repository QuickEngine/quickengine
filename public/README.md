# public/

Public-facing documentation lives here.

This directory is **tracked in git** and intended for content that can be shared outside the
company: developer docs, guides, API references, quickstarts, and marketing-supporting material.

It is intentionally empty for now. Public documentation is written **after** the deferred
UI/UX, billing, and tier decisions are made together, so that what we publish matches the real,
shipped product. Until then, all working notes and internal planning live in the git-ignored
`internal/` directory, never here.
